# 新手环境准备

你目前没有 Mac 和 Xcode，这完全没问题——本项目的设计就是让 GitHub 的云端 Mac 帮你编译，你只需要一台能上网的电脑（Windows/Mac 都行）用来编辑文字文件、提交代码。

## 你需要准备/确认的东西

| 项目 | 说明 |
|---|---|
| GitHub 账号 | 你已经有了 |
| GitHub 私有仓库 | 你已经有了，把本项目文件夹的内容推送进去 |
| Apple ID | 你已经有了 |
| Apple 开发者账号（免费） | 你已经有了。免费账号意味着：只能签发 7 天有效期的开发证书，无法使用 TestFlight / Ad Hoc 正式分发，只能装到你手动在 Apple Developer 后台注册过 UDID 的设备上。如果以后想长期稳定使用，建议升级到 99 美元/年的付费账号。 |
| iPhone | 用来最终安装 IPA、注册 UDID |
| 一个文本编辑器 | 推荐 [VS Code](https://code.visualstudio.com/)，装一个 Swift 语法高亮插件就够用了，你不需要用它编译，只是方便看代码、改配置 |
| Git 工具 | Windows 可以装 [Git for Windows](https://git-scm.com/download/win)，或者直接用 GitHub Desktop |

## 关于"没有 Mac 也能开发 iOS"这件事

这不是权宜之计，而是完全可行的正规做法：
- 项目源码用任何电脑上的文本编辑器编辑
- `git push` 到 GitHub
- GitHub Actions 会自动启动一台苹果云端 Mac（`macos-26` 镜像，自带 Xcode 26），按 `.github/workflows/ios-build.yml` 的步骤编译
- 编译产物（IPA）会出现在 Actions 运行记录的 "Artifacts" 里，可以直接下载

唯一的限制是：你没法在本地实时预览界面效果（SwiftUI Preview 需要 Xcode），只能靠"改代码 → 推送 → 等 Actions 跑完 → 看结果"这个循环来验证效果，每次大概几分钟。如果以后觉得这样效率太低，可以考虑短期租用云端 Mac（如 MacinCloud）或者购入一台二手 Mac mini。
