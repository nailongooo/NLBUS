# 项目总览

这是一个类似"掌上公交"的出行到站提醒系统，包含三个部分：

```
transit-tracker/
├── ios/            iOS App 源码（SwiftUI，用 XcodeGen 管理工程文件）
├── backend/        后端服务（Node.js + Express + MySQL），部署在你自己的云服务器上
├── admin-web/       网页版管理后台（纯静态 HTML/JS，可以直接丢进 Nginx 里托管）
└── .github/workflows  GitHub Actions 配置，负责在 GitHub 的 Mac 云主机上编译打包 IPA
```

## 建议的上手顺序

1. 读完这份总览
2. 看 `01-新手环境准备.md`，了解要注册/准备哪些东西
3. 看 `03-后端部署教程.md`，先把后端部署起来（因为 App 需要连它）
4. 把 `ios/App/Services/APIClient.swift` 和 `admin-web/js/api.js` 里的 `YOUR_DOMAIN_HERE` 换成你自己的域名
5. 把整个项目推送到 GitHub 私有仓库
6. 看 `02-GitHubActions证书打包教程.md`，配置好签名相关的 Secrets（这一步可选，不配置也能看到编译成功）
7. 到 GitHub 仓库的 Actions 页面查看构建结果，下载生成的 IPA

## 已经确认的产品逻辑（方便你回忆当时的沟通结果）

- 到站提醒规则：距目标站 1500 米预提醒、500 米即将到站、150 米到站提醒，均需要连续 2 次定位确认，GPS 精度差于 65 米时不参与判断。
- 用户点击"开始行程"才会真正启用后台定位，行程结束立即关闭。
- 车辆实时位置采用众包方案：正在乘车的用户会匿名上报位置，服务器聚合展示给同路线的其他乘客，不依赖任何第三方地图公司的公交数据接口。
- App 默认不强制登录，所有行程历史只保存在手机本地；账号体系是可选的加分项，用于以固定昵称提交公开路线。
- 免费 Apple 开发者账号的能力边界（证书 7 天、无法用 TestFlight/Ad Hoc 正式分发）已经在代码和 CI 配置里如实考虑，见 `02-GitHubActions证书打包教程.md` 里的说明。

## 关于代码质量的诚实说明

我在生成这批代码的沙盒环境里没有 Mac、没有 Xcode，无法真正执行 `xcodebuild` 编译一遍。所有 Swift 代码都经过了仔细的人工审查（语法、类型、iOS API 可用性），但**第一次真正的编译验证会发生在你推送到 GitHub 之后，由 Actions 里的 Xcode 完成**。如果 Actions 报错，把日志片段发给我，我可以针对性地定位和修复，这比空泛地说"我测试过了没问题"更负责任。
