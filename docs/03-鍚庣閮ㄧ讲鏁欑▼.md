# 后端部署教程

假设你已经有：一台云服务器（Linux，推荐 Ubuntu 22.04）、一个域名、HTTPS 证书、MySQL。下面以 PM2 直接运行的方式为主（不强制要求 Docker），Docker 方式在文末补充。

## 1. 准备服务器环境

SSH 登录你的服务器：

```bash
# 安装 Node.js 20（用 nvm 管理版本更省心）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
node -v   # 确认是 v20.x

# 安装 PM2（用来让 Node 服务常驻后台运行、开机自启）
npm install -g pm2
```

## 2. 上传/拉取代码

```bash
cd /opt
git clone <你的仓库地址> transit-tracker
cd transit-tracker/backend
npm install --omit=dev
```

## 3. 配置数据库

用你已有的 MySQL：

```bash
mysql -u root -p < ../backend/sql/schema.sql
```

这会创建 `bus_tracker` 数据库和所有需要的表。然后新建一个专门给后端用的数据库账号（不要用 root）：

```sql
CREATE USER 'bus_tracker'@'localhost' IDENTIFIED BY '设置一个强密码';
GRANT ALL PRIVILEGES ON bus_tracker.* TO 'bus_tracker'@'localhost';
FLUSH PRIVILEGES;
```

## 4. 配置环境变量

```bash
cp .env.example .env
nano .env
```

按你的实际信息填好 `DB_HOST` / `DB_USER` / `DB_PASSWORD` / `DB_NAME`，`JWT_SECRET` 换成一段随机字符串（可以用 `openssl rand -hex 32` 生成一个）。

## 5. 创建管理员账号

```bash
node scripts/create-admin.js admin 你自己设置的密码
```

## 6. 用 PM2 启动服务

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # 会提示你执行一条命令，复制粘贴执行一下，实现开机自启
```

确认服务正常：

```bash
curl http://127.0.0.1:3000/api/health
# 应该返回 {"status":"ok", ...}
```

## 7. 配置 Nginx 反向代理 + HTTPS

参考 `backend/nginx/bustracker.conf.example`，把里面的域名和证书路径换成你自己的，然后：

```bash
cp backend/nginx/bustracker.conf.example /etc/nginx/conf.d/bustracker.conf
nano /etc/nginx/conf.d/bustracker.conf   # 修改域名和证书路径
nginx -t                                  # 测试配置语法
systemctl reload nginx
```

## 8. 部署管理员网页后台

`admin-web/` 是纯静态文件，最简单的方式是直接丢到服务器上让 Nginx 托管：

```bash
mkdir -p /var/www/bustracker-admin-web
cp -r admin-web/* /var/www/bustracker-admin-web/
```

Nginx 配置示例里已经包含了 `/admin/` 这个 location，指向这个目录。部署完成后访问
`https://你的域名/admin/index.html` 应该能看到登录页面。

**别忘了**把 `admin-web/js/api.js` 和 `ios/App/Services/APIClient.swift` 里的 `YOUR_DOMAIN_HERE` 替换成你自己的真实域名，然后重新部署/重新打包。

## 9. 定期清理众包定位数据（隐私保护 + 防止表变大）

```bash
crontab -e
```
加一行，每 10 分钟清理一次超过 30 分钟的数据：
```
*/10 * * * * cd /opt/transit-tracker/backend && /root/.nvm/versions/node/v20.*/bin/node scripts/cleanup-old-pings.js >> logs/cleanup.log 2>&1
```

## 10. 验证整个链路

```bash
curl https://你的域名/api/health
curl https://你的域名/api/routes
```

如果都能正常返回 JSON，后端就部署完成了。

---

## 附：用 Docker 部署（可选）

如果你更习惯 Docker：

```bash
cd backend
cp .env.example .env   # 同样填好数据库信息（DB_HOST 填你服务器上 MySQL 实际能被容器访问到的地址）
docker compose up -d --build
docker compose logs -f
```

`docker-compose.yml` 默认只启动后端本身、连接你已有的 MySQL；如果想让 MySQL 也跑在 Docker 里，文件里有注释掉的部分可以参考启用。
